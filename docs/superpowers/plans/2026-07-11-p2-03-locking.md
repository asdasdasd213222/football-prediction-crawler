# P2-03 Distributed Locking Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent overlapping source tasks with renewable Redis leases while
allowing recovery after worker loss.

**Architecture:** `locking.py` owns atomic Redis lease operations and does not
know Celery or sources beyond a site-neutral id. Celery task entry points wrap
their execution in a lease and log an explicit `skipped_overlap` outcome.

**Tech Stack:** Python 3.12, redis-py, Celery, pytest, Docker Compose.

## Global Constraints

- Store timestamps as Beijing local time (`Asia/Shanghai`).
- Use no credentials, production deployment, source-specific logic, or P2-04
  retry/rate-limit/circuit-breaker behavior.
- Run Ruff, Mypy, Pytest, Docker Compose config, and local Redis acceptance.
