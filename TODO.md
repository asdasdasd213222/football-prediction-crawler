# 多网站实时爬虫项目 TODO

> 目标：依托 Codex 开发和维护一个稳定、可观测、可扩展的多网站数据采集系统。
> 生产抓取由独立调度器和 Worker 执行；Codex 负责规划、编码、测试、审查、故障分析和适配器维护。
> 默认最低抓取周期：60 秒。优先使用官方 API、Webhook、WebSocket、SSE 或 RSS；普通网页采用轮询与变化检测。

---

## 使用规则

- [ ] 每次只让 Codex 执行一个阶段或一个边界明确的任务。
- [ ] Codex 修改代码前必须先阅读 `AGENTS.md`、本文件和相关源码。
- [ ] Codex 开始复杂任务前必须先给出实施计划。
- [ ] 不允许仅生成占位代码后宣称完成。
- [ ] 不允许跳过测试、类型检查和静态检查。
- [ ] 不允许把账号、Cookie、Token、代理密码或数据库密码提交到 Git。
- [ ] 不允许实现 CAPTCHA 绕过、登录绕过、付费墙绕过或访问控制规避。
- [ ] 不允许未经人工审核直接部署 Codex 自动生成的生产变更。
- [ ] 一个任务只有在对应验收项全部通过后才能勾选。
- [ ] 每完成一个任务，更新本文件中的状态，并在提交信息中引用任务编号。

### 统一完成标准

每次代码变更至少满足：

```bash
ruff check .
ruff format --check .
mypy src
pytest -q
docker compose config
```

涉及数据库时还必须满足：

```bash
alembic upgrade head
alembic downgrade -1
alembic upgrade head
```

涉及容器或运行时逻辑时还必须完成一次本地端到端验证。
